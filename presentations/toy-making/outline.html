Making Toys

Why this talk/event?

 A few week ago, I tried an experiment

  I recorded an animated GIF coding

   I simply wanted to see how it looked on Twitter

   I was initially scolded for posting large animated GIFs

   Then people started retweeting it

  I created an account

   Called it @fsharpcasts

   Posted more simple videos

   @dsyme retweeted it

   Got 300 followers over the weekend

  I tried more experiments

   Recorded longer videos doing more interesting things

   Recorded short videos that minimally demo tools

   People overwhelmingly preferred the small + simply demos

  I embarassed myself

   Demoed code that is vulnerable to XSS

   Implemented binary tree inversion

   Snarkily verified it worked with FsCheck after it was challenged

   Discovered my type was inherently wrong

   Posted the fix in an effort to lemons into lemonade

  I had a lot of fun

   Talked with people I typically don't

   Coded more for fun

   Started looking through my old notes

   Dredged up half-completed projects

   So I can share them with you

   And resurrect some ideas

 It's about fostering a fun community

  Encouraging making things

   Even if they're temporarily bad

  Making stuff for and by us

   Tools

   Libraries

   Demos

  Sharing ideas

   Giving Constructive Feedback

   Seeding ideas

  Making friends

 The ego, banish it!

How Playing Differs from Prototypes

 The objective is fun and learning something

 Pick a direct, don't map a plan

  Compasses over maps

  Pivoting happens a lot faster

  Allows for serendipitous fun

 Sharing make it better

  Start from the end and fill in backwards

  Prototype with crude instruments

   Paper taped to a cardboard iPhone

 Incomplete ideas are cherished

  They're catalysts for better ideas

  They invite sharing

 Keep it informal

  It encourages participation
